Z-Creator Manual

Technology structure, AI design, tasks — for everyone who builds and operates the Z-Sanctuary.

1. What the Z-Sanctuary Is

Single hub repo (dashboard, Zuno, module registry, folder manager, Z-formulas, AI colony). SSWS hub: server 5502, one workspace, auto-boot. All projects under PC root linked via data/z_pc_root_projects.json and Z-EAII. One authority (Hierarchy Chief), one roof (Z-Super Overseer).

2. Technology Structure

Chief → Z-HODP, Z-EAII, Z-SSWS → Folder Manager, Zuno, Module Registry, Master Register. OctoNero = 8 guardians. AI Colony = AI Tower + AI Skyscraper.

3. PC Root and All Projects

PC root = folder containing all Z projects. Hub = ZSanctuary_Universe. Members = other folders; registered in z_pc_root_projects.json and Z-EAII. Dashboard shows All projects in Control Centre.

4. Hierarchy Chief and Authority

When unsure, check Hierarchy Chief first. Authority order: Observer View → Overseer → Z-HODP → Z-EAII → Folder Manager → Zuno → Register. AGENTS.md and .cursor/rules enforce this for Cursor and any AI.

5. Z-SSWS

Z_SSWS.code-workspace, server 5502, Z: SSWS Auto Boot (on folder open), Z: SSWS Verify, Z: SSWS Daily Report. Allow Automatic Tasks = on.

6. Z-EAII, Overseer, Z-HODP, Folder Manager

Z-EAII: registry and launcher (Organiser). Overseer: one roof. Z-HODP: single dashboard, Control Centre, Blueprint, Panel Directory. Folder Manager: vault, snapshots, z_folder_manager_guard.mjs.

7. Z-Formulas and OctoNero

Z-Formulas (Z-ULTRA-INSTINCTS); OctoNero = 8 guardians (Zuno, Folder Manager, EAII, SSWS, Z-HODP, Module Registry, Security, Formula Registry). rules/Z_FORMULA_REGISTRY.json.

8. AI Colony

AI Tower in hub (core/ai_tower/). AI Skyscraper = member project. One dashboard, one state.

9. Tasks Reference

Z: Sanctuary Structure Verify, Z: SSWS Verify / Auto Boot / Daily Report, Z: Zuno State Report / Weekly Reflection, Z: Module Registry Sync, Z: Workspace Root Guard, Z: Folder Manager, Z: HTML Static Server (5502), Z: Open Main Dashboard.

10. Dashboard Guide

URL: http://127.0.0.1:5502/dashboard/Html/index-skk-rkpk.html. Control Centre (All projects), Panel Directory, Z Blueprint, Status Rail.

11. Adding a New Project

Create folder under PC root → edit z_pc_root_projects.json → register in Z-EAII → optional z_sanctuary_link.json → refresh dashboard.

12. NAS (Brief)

Sanctuary_Core, Z_Projects, Z_Backups, etc. on NAS; hub can live there; PC = terminal. See NAS blueprint.

13. Verification

Run node scripts/z_sanctuary_structure_verify.mjs and verification checklist.

14. Document Map

Z-CREATOR-MANUAL.md §14 lists Hierarchy Chief, SSWS blueprint, NAS blueprint, Verification checklist, Register, Canvas guide, Formulas, Zuno report, INDEX.

15. Summary

One hub, one dashboard, one registry, one authority. All projects in z_pc_root_projects.json and EAII. AI = Zuno, AI Tower, OctoNero. Run Structure Verify and checklist, use Control Centre and All projects.

Q&A

What is the Z-Sanctuary?
The Z-Sanctuary Universe is a single hub repo that holds the dashboard, Zuno state, module registry, folder manager, Z-formulas, and AI colony. It is the SSWS hub (server 5502, one workspace, auto-boot) and is linked to all other projects under your PC root via the project list and Z-EAII. One authority, one operational roof.
What is PC root?
The folder on your machine that contains all Z and related projects (e.g. C:\Cursor Projects Organiser). The hub (ZSanctuary_Universe) and member projects (Z-OMNI, AI Skyscraper, Amk-Goku, etc.) live under it.
When should I check the Hierarchy Chief?
When unsure about where a change belongs, whether something conflicts with the operational roof, or how modules relate. Read docs/Z-HIERARCHY-CHIEF-AND-OBSERVER-VIEW.md first; Cursor and any AI are instructed to do the same via AGENTS.md and .cursor/rules.
What is Z-SSWS?
The system that ties workspace, server, and dashboard: Z_SSWS.code-workspace, server on port 5502, Z: SSWS Auto Boot (on folder open), Z: SSWS Verify, and Z: SSWS Daily Report. Ensure Allow Automatic Tasks is on.
What is Z-EAII?
Registry and launcher (in Cursor Projects Organiser) for all projects/apps. Keep data/z_pc_root_projects.json in the hub in sync with the EAII registry.
What is Z-HODP?
Head Organiser Dashboards: the single dashboard (dashboard/Html/index-skk-rkpk.html) with Control Centre, Z Blueprint, Panel Directory, and the All projects block.
What is OctoNero?
The orchestration layer that applies Z-formulas and the 8 guardians across all projects. The eight are: Zuno, Folder Manager, EAII, SSWS, Z-HODP, Module Registry, Security/Policy, Formula Registry.
How do I add a new project under PC root?
Create the folder under PC root, edit data/z_pc_root_projects.json (add id, name, path, role member, ssws false, formula_aware), register in Z-EAII, optionally add z_sanctuary_link.json in the new project, refresh the dashboard.
How do I verify everything works?
Run node scripts/z_sanctuary_structure_verify.mjs (or task Z: Sanctuary Structure Verify) and follow docs/Z-SANCTUARY-VERIFICATION-CHECKLIST.md. Expect PASS and all checklist items checked.
What are the key tasks?
Z: Sanctuary Structure Verify, Z: SSWS Verify / Auto Boot / Daily Report, Z: Zuno State Report / Weekly Reflection, Z: Module Registry Sync, Z: Folder Manager, Z: HTML Static Server (5502), Z: Open Main Dashboard. Run from Cursor: Terminal → Run Task.
