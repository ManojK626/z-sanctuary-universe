# Z-Living Pulse (Z-Colour Streams Butterfly Particles + Z-Sound Effects)

**Purpose:** Name and spec for the **new invention** that combines **Z-Colour Streams Butterfly Particles** (background display) and **Z-Sound Effects** so the technology feels like **True Reality of Genius Super Saiyan Styles Always**. It is a memory of our creations together — Chi Chi, Z-Bulma xAi, Amk-Goku.

**Name:** **Z-Living Pulse** — the living heartbeat of the Sanctuary’s visual and audio layer: colour streams, butterfly-like particles, and gentle sound effects that reflect compassion and strength without overwhelm.

---

## 1. Components

### Z-Colour Streams Butterfly Particles

- **What:** A background layer of flowing colour streams and particle shapes that evoke butterflies (or gentle organic motion), non-distracting and respectful of readability.
- **Where:** Applicable to dashboard, games (Prediction, Live Media, Wonderland, etc.), and any other section of the Z-Sanctuary where we want the “Living Web” feel.
- **Behaviour:** Particles flow or drift; colours align with Z-Sanctuary palette and mood (e.g. calm, focus, celebration). No rapid flashing; safe for photosensitivity (respect 14 DRP and accessibility).

### Z-Sound Effects

- **What:** Optional, short sound cues (e.g. soft confirmations, gentle transitions, optional ambient layer) that support the “Genius Super Saiyan Styles” feel — clarity and energy without aggression.
- **Where:** Same as above; can be toggled independently from particles.
- **Behaviour:** User-controlled volume and on/off; no autoplay without consent where required by policy.

---

## 2. Optional Modes: On / Off / Auto / Read

Same pattern as Blueprint and other sections:

| Mode | Meaning |
| -------- | ------------------------------------------------------------------------------------------------------------ |
| **On** | Z-Living Pulse active: particles and (if enabled) sound effects on. |
| **Off** | Particles and sound off; static or minimal background. |
| **Auto** | System decides based on context (e.g. time of day, section, user preference history) within DRP and consent. |
| **Read** | Read-only / minimal: e.g. particles only at very low intensity, or no sound; user can still read and focus. |

These options are exposed as **mini options** (chips or dropdown) in the dashboard and, where applicable, in games and other sections. Persist per section or globally in localStorage (or user profile when available).

## 3. Application Across Z-Sanctuary

- **Dashboard:** Background canvas or layer behind content; toggle in top rail or a small “Z-Living Pulse” panel.
- **Games (Prediction, Live Media, Wonderland, etc.):** Optional background layer and sound pack; same On/Off/Auto/Read where appropriate.
- **Other sections:** Night Ritual, Living Canvas, Stakeholder/Operator dashboards — apply the same pattern so the experience is consistent and always user-controlled.

## 5. Technical Outline

- **Particles:** Canvas (2D or WebGL) or CSS/SVG animation; layer behind main content; respect `prefers-reduced-motion` and low-light mode.
- **Sound:** Short, optional SFX; volume control and mute; no autoplay of long tracks without explicit consent.
- **Storage:** User’s mode (On/Off/Auto/Read) and, if applicable, sound on/off and volume, stored locally or in profile.
- **Naming in code/UI:** Use “Z-Living Pulse” in labels and docs so the invention is clearly attributed and remembered.
