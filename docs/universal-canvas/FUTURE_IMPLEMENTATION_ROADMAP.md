# Z-UCCR — Future Implementation Roadmap

**Purpose:** Phase boundaries for Universal Canvas engineering. **No phase below is mandated by doctrine files alone**—each needs human go / ACN-MTEH when code ships.

---

## Phase Z-UCCR-0 — Doctrine only (current)

- Charter + panel/guardian/ethics/a11y/roadmap markdown complete.
- No required new UI in repo from doctrine.

---

## Phase Z-UCCR-1 — Static mock / wireframe

- Figma or static HTML wireframe (optional sandbox path) showing layout regions and panel placeholders.
- No production wiring to live data APIs.

---

## Phase Z-UCCR-2 — Read-only data panels

- Panels consume **offline JSON snapshots** or static fetch of existing report files served locally.
- No writes via canvas actions.

---

## Phase Z-UCCR-3 — Operator workflow launcher

- Copy-to-clipboard / deep links to sanctioned VS Code/Cursor tasks and docs.
- Still no unattended execution.

---

## Phase Z-UCCR-4 — Guardian / audit visualization

- STOP_GATE, amber/hold truthful rendering tied to verifier JSON.
- Human-readable drill-down receipts.

---

## Phase Z-UCCR-5 — Controlled assistant routing

- Mini-bot doctrines surfaced as interactive routers with **confirmation** modals gates.
- Autonomy expansion gated per [ETHICS_AND_CONSENT_GATES.md](ETHICS_AND_CONSENT_GATES.md).

---

## Phase Z-UCCR-6 — Optional public-safe trust view

- Highly redacted “Sanctuary trust posture” page for partners—**no** secret metrics, **no** vault, **no** user data.
- Separate legal review artifact before enable.

---

## Exit criteria per phase

Each phase closes with: accessibility review note, consent checklist tick, receipt path, and **no** regression of existing pause/enforcer discipline.
