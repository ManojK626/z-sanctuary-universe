# Z-UCCR — AI Core Visualization Requirements

**Purpose:** Align **tower / colony / fusion** metaphors with **honest posture display**—no fake “hive mind online” dashboards.

---

## 1. Goals

- Show **declared AI surfaces** from registry/register (names, tiers, stewardship).
- Indicate **verification state** tied to sanctioned tasks (conceptual—not live GPU temps).
- Support **routing** metaphors compatible with MiniAI doctrine (observe → suggest → escalate).

---

## 2. Allowed elements

- **Core nodes:** Overseer umbrella, EAII doorway, mirrors, tower deck labels from blueprints.
- **Status ribbons:** Planned / stubs / guarded / gated.
- **Pointers:** Links to docs (Super Chat blueprint, SSWS colony doc).
- **Compassion strip:** Quiet reminder—DRP-aligned—when risk band elevated.

---

## 3. Forbidden elements

- Real-time deceptive “confidence %” invented by UI.
- Showing model weights or private prompt banks.
- **Auto-spinning** cognition animations as default hype (see accessibility doc).
- Buttons that enqueue agent mesh jobs without confirmation.

---

## 4. Trust colors

Reuse trust signal palette from [VISUAL_EFFECTS_AND_THEMES.md](VISUAL_EFFECTS_AND_THEMES.md). Ambiguity renders as **muted neutral**, not flashy “unknown-danger red.”

---

## 5. Phase

**Z-UCCR-2:** static snapshot of tower layout. **Z-UCCR-5:** controlled assistant routing overlays with explicit confirmations.
