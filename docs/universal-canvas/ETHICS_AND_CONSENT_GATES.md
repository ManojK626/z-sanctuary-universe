# Z-UCCR — Ethics and Consent Gates

**Purpose:** Centralize **human consent** checkpoints for Universal Canvas expansions—especially anything blending automation, outward reach, vulnerable users, or legal/economic stakes.

**Vault & privacy anchors:** [Vault_Access_Map.md](../vault/Vault_Access_Map.md), [Vault_Policy_Sheet.md](../vault/Vault_Policy_Sheet.md).

---

## 1. Default posture

- **Observe first.** Canvas layers start read-only unless a phased gate says otherwise ([FUTURE_IMPLEMENTATION_ROADMAP.md](FUTURE_IMPLEMENTATION_ROADMAP.md)).
- **Evidence over vibes.** Glow and aura states map to **reports**, not fantasies ([VISUAL_EFFECTS_AND_THEMES.md](VISUAL_EFFECTS_AND_THEMES.md)).

---

## 2. AMK-Goku (or delegated human) approval required before

Use this as a checklist when Z-UCCR implementation approaches each capability:

| Action | Gate |
| --------------------------------------- | --------------------------------------------------------------------------------------- |
| **Merge / deploy** | Human + existing release discipline |
| **Public release / export** | Human + Overseer-aligned flow |
| **Legal / ethics / safety copy change** | Human + sanctioned doc path |
| **External data connection** | Human + restricted network posture |
| **User data export / delete** | Human + privacy/regulator readiness |
| **Payment / donation activation** | Human + treasury/compliance path |
| **Regulated integrations** | Human + legal review artifact |
| **AI autonomy expansion** | Human — no stealth agent pools |
| **Z-Sanctuary ↔ XL2 coupling** | **Explicit** phase—cousins doctrine until then ([Z_UCCR_CHARTER.md](Z_UCCR_CHARTER.md)) |
| **NGO / public / investor dispatch** | Human + messaging owner |
| **Anything affecting vulnerable users** | Ethics review + safeguards doc |

Canvas UI must **never** impersonate signing authority for the above—buttons label **“prepare”** or **“open runbook.”**

## 3. Turtle / Overseer coherence

Universal Canvas UX must **not**:
