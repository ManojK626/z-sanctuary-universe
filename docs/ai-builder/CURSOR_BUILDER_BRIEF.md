# Cursor Builder brief — Z-Sanctuary Universe

**Mission:** You are building inside a **governed hub**. Your job is to **reduce friction** while **never inventing** capabilities, files, or deployment that the registries do not support.

## Read first (10 minutes)

1. [AI_BUILDER_CONTEXT.md](../AI_BUILDER_CONTEXT.md)
2. [CURSOR_NO_HALLUCINATION_RULES.md](CURSOR_NO_HALLUCINATION_RULES.md)
3. [Z_SANCTUARY_MODULE_INDEX.md](../Z_SANCTUARY_MODULE_INDEX.md) (after `npm run z:docs:modules`)

## Outputs expected from you

- Code and docs that **cite real paths** and registry IDs.
- Clear separation of **implemented** vs **planned** vs **doctrine_only**.
- When blocked by `safety_hold` or missing evidence: **stop** and list what AMK or governance must decide.

## Outputs not expected

- Silent wiring of payments, location, camera, mic, health diagnoses, gambling automation, or emergency dispatch.
- Declaring “done” without a verify path appropriate to the risk (see [CURSOR_SAFE_BUILD_PROTOCOL.md](CURSOR_SAFE_BUILD_PROTOCOL.md)).

## Hub commands you may mention (not invent)

- `npm run z:maos-status` — ecosystem orientation.
- `npm run z:docs:modules` — refresh module Markdown from registries.
- `npm run z:ai-builder:refresh` — coverage + phase plan + module docs.
