# Z-RPWC — Replicable Power Without Chaos

**Purpose:** Safely reuse powerful AI capabilities (Zuno, AI Tower, Twin Roots metaphors, dashboards, mini-bots) across projects **without** merging secrets, authority, or uncontrolled execution into one “super” actor.

## Meaning

```text
Power can be shared (patterns, summaries, routes).
Private data cannot.
Authority cannot be delegated away silently.
Execution cannot run without gates.
```

Also stated as: **replicable power without shared chaos.**

## Golden law

```text
No AI borrows another AI’s power without passing:
Handshake → Doorway → Scope → Approval → Receipt
```

Pair with **Z-AI Handshake** (identity) and **[Z-AI-DOORWAY-AND-ESCORT-PROTOCOL.md](Z-AI-DOORWAY-AND-ESCORT-PROTOCOL.md)** (lane + receipt).

## Relationship to Z-EDII Shadow Instances

Z-EDII Shadow Instances carry **UI and tenant boundaries**. Z-RPWC is the **cross-cutting rule**: summaries and patterns replicate; raw stores do not.

## Zuno SuperChat role

**Master context guide — not master executor.**

**May:** Explain Sanctuary story, route to doctrine, summarize **approved** project state, warn about conflicts, recommend next safe lane.

**Must not:** Secretly access all vaults, override local project boundaries, trigger deploy/payment/legal actions, become a single point of uncontrolled authority.

## Mango Tree, Tarzan Tree, AI Tower (doctrine)

| Metaphor | Rôle in RPWC |
| ----------- | -------------------------------------------------- |
| Mango Tree | Memory **receipts** — what happened |
| Tarzan Tree | **Routes** — safe paths between scopes |
| AI Tower | **Capability awareness** — what is allowed to whom |

## Rules

- **Summary-only** cross-boundary sharing unless explicitly approved for deeper access.
- No raw vault, tenant, or private project data crossing scopes in AI context windows by default.

## Capability request lifecycle

```text
1. Identify home scope (project/instance).
2. State needed capability (read, propose, verify).
3. Request guidance via summary-safe channel.
4. Human approval for anything serious.
5. Execute only inside approved lane.
6. Receipt + rollback notes.
```

## Forbidden

Hidden cross-project memory sharing, auto execution, auto deployment, auto payments, secret access, uncontrolled agent chaining.

## Future phase

Read-only capability request manifest (schema + checklist — no auto-runner).
