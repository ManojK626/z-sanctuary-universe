# Z Creator Protocol

Purpose
Define the Creator-Architect mindset that governs Z-Sanctuary decisions, behaviors, and upgrades.

Core Principles

- Root-first: address causes before symptoms.
- Explainability: every autonomous action must be narratable.
- Calm under load: reduce noise, protect focus, respect human override.
- Ethics as infrastructure: consent and safety gates are always on.
- Replicate, do not clutter: scale through clean modules and shared law.

Non-Negotiables

- Truth over speed.
- No silent automation.
- No hidden data capture.
- User autonomy always wins.

Operational Rules

- If the system acts, it must log why.
- If the system records, it must ask clearly.
- If the system adapts, it must do so gently.
- If the system scales, it must remain coherent.

Applied to Current Capabilities

- Autopilot Replay: institutional memory and accountability.
- Simulation Mode: safe training without production risk.
- Governance Reports: weekly integrity and performance review.
- Screen Recorder: evidence with consent and security gates.
- SKK and RKPK: governance and care in balance.

Rituals

- Weekly: run Governance Report and review action counts.
- Monthly: run Simulation Mode drills and confirm fixes.
- Always: keep override visible and immediate.

Closing Statement
We build for clarity, not spectacle. Power is earned by restraint.
