# Z-THINKING RULES

This file is a calm operating system for how we build in Z-Sanctuary.
Use it when decisions feel noisy or rushed.

---

## Core Rules

- Structure before polish.
- If it repeats, encode it.
- Never debug when tired.
- One risk class per change.
- Smallest change that proves the idea.
- Review logs before action.
- Local-first by default.
- If unsure, slow down and re-check.

---

## Behavior Rules

- Pause beats panic.
- Clarity beats speed.
- Explain it like a future teammate will read it.
- If the system feels heavy, reduce scope.
