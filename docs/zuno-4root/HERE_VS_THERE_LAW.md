# Here vs there law (evidence boundary)

## Here (allowed claims)

- What exists **in this repo**: paths, scripts, registry rows, reports generated by named commands.
- What operators **ran** and can paste output for.

## There (not claimed from dashboard alone)

- Production traffic, revenue, user counts, or hosted health **without** corresponding report or API evidence.
- “Everything is green” **without** citing verify artefacts.

## ZUNO-4ROOT application

The four-root dashboard view is **symbolic + selectively live**. Live portions (e.g. registry-derived completion in companion canvas when fetch succeeds) must degrade gracefully to **doctrine-only** text when data is missing — never placeholder metrics presented as measured truth.
