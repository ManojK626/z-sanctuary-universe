# Signals and gates (ZUNO-4ROOT)

## Signal categories (conceptual)

| Label | Meaning |
| ----------------- | --------------------------------------------------------- |
| `USER_INPUT` | Operator or creator intent captured in tools or dashboard |
| `AI_SIGNAL` | Model or bot output suggesting an action |
| `SYSTEM_HEALTH` | Verify scripts, audits, registry posture |
| `14DRP_CHECK` | Human compassionate governance ritual |
| `SKK_RKPK_REVIEW` | Guardian framing on dashboard / checklist |
| `STOP_GATE` | Hard pause — policy, consent, or risk conflict |
| `HUMAN_APPROVAL` | Explicit AMK / governance OK for sacred move |
| `AUDIT_LOG` | Written receipt (report, JSON, Markdown) |

## Gates

- **Soft gate:** documentation or lint must pass before merge suggestion.
- **Hard gate:** execution enforcer, manual release, consent matrix — human authority.

## No fabrication

Dashboard badges may mirror these labels **doctrinally**. Live counts (e.g. STOP_GATE totals) appear **only** when backed by real stored metrics — otherwise show explanatory text, not fake numbers.
