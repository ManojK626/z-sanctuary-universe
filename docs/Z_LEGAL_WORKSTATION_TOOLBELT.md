# Z-LEGAL-TOOLBELT-1 — Lawyer Workstation Tool Dock

## Purpose

Define a compact legal toolbelt for the lawyer workstation so workflows are clear and professional without opening runtime legal lanes.

This phase is UI metadata plus governance doctrine only.

## Tool dock structure

Use one compact dock:

- Left: tool categories
- Center: selected viewer/panel
- Right: evidence/risk/context drawer
- Bottom: receipts and export draft queue

Avoid panel sprawl and keep legal flow focused.

## Tool categories (Phase 1 posture)

- Document Viewer — read-only
- Image/Video Evidence Viewer — no AI analysis runtime
- Timeline Builder — metadata only
- Extract/OCR placeholder — future BLUE lane only
- Redaction Checklist — no destructive edit
- Notebook — local/manual later
- Citation/Source Binder — read-only
- Export Pack Builder — draft concept only
- Risk Queue — dashboard metadata
- Contract/IP Tracker — no filing/contract claim
- Client Consent Tracker — no client-data intake
- AI Assistant Panel — no legal advice
- Canvas/World Map — visual-only

## Boundaries

- No uploads
- No client-data intake
- No OCR runtime processing
- No video analysis runtime
- No legal advice runtime
- No e-signature runtime
- No court filing
- No payments
- No external APIs
- No deployment

## Locked law

- Toolbelt != live case system.
- Viewer != evidence intake.
- OCR placeholder != document processing.
- Notebook != privileged storage.
- AI assistant != legal adviser.
- Export draft != court filing.
- GREEN != launch.
- AMK-Goku owns sacred moves.
