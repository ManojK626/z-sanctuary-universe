# Magical visual entitlement boundary (ZMV-1A)

## Rule

**Presentation kits are not product access.** Shared CSS or HTML mood does not grant accounts, API keys, paid tiers, or synchronized cloud state from ÉirMind or any sibling project.

## Separation table

| Concept | Owned by | Hub registry role |
| -------------------- | -------------------------------------- | ------------------------------------------- |
| SKUs / prices | Source project (e.g. ÉirMind) | Never auto-imported |
| Entitlement | Source project + formal bundle charter | Declared in ZSX catalogs |
| Visual patterns | Attributed design lineage | `z_magical_visual_capability_registry.json` |
| Z-QUESTRA PlayGarden | z-questra | Non-predatory local rules still apply |

## Shared CSS does not merge SKUs

Duplicating or referencing `sanctuary-magical-bridge.css` (when present) does **not**:

- Create a storefront line item,
- Unlock ÉirMind backend features,
- Imply a bundle deal unless a **bundle charter** exists (see `data/z_service_entitlement_catalog.json`).

## Shadow previews

Copied UI patterns must show **source project**, **status**, and **not-live** posture per shadow policy. Pricing and access remain with the **pricing_owner** on each registry row.

## Escalation

Live embed, iframe runtime coupling, shared storage, or API use requires:

1. Written charter
2. **14 DRP** (safety / harm reduction)
3. **14 DOP** (operational discipline / sign-off)
