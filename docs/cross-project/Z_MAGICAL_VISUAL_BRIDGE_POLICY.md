# Z Magical Visual Bridge — policy (ZMV-1A)

## Purpose

Register **ÉirMind / AMI / Sister Aisling-Sol Magical AI** visual and canvas patterns so Z-Sanctuary can **reuse beauty and clarity** without merging products, SKUs, prices, or live services.

Canonical registry: **`data/z_magical_visual_capability_registry.json`**.

## Golden law

```text
Visual reuse does not grant service access.
Capability reference does not grant entitlement.
Entitlement does not grant billing permission.
Billing permission does not grant live bridge.
Live bridge requires charter plus 14 DRP and 14 DOP review.
```

## What is shared

| Layer | Shared how |
| --------------- | --------------------------------------------------------- |
| CSS / HTML mood | Docs link, static reference, manual copy with attribution |
| Motion / hero | Same, plus reduced-motion respect |
| Tokens / chroma | Docs-first alignment with Z-VIL |

## What is not shared (without charter)

- Live embeds, shared user storage, API bridges, cloud sync
- Automatic SKU or tier inheritance from ÉirMind into hub or Z-QUESTRA
- Billing execution or entitlement enforcement from this metadata

## Attribution (shadow / preview)

When another surface mirrors this visual language:

```text
Inspired by ÉirMind Magical AI visual bridge.
Local preview only.
No ÉirMind paid service access is included.
```

Follow **`docs/cross-project/Z_SHADOW_PREVIEW_POLICY.md`** and **`data/z_shadow_preview_policy.json`**.
