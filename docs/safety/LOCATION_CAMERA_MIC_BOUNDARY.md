# Location, camera, and microphone boundary

**Posture:** hardware-adjacent features require explicit consent and narrow purpose.

## Location

- **Default off** until a clear, in-product consent explains **why** location is needed.
- **Emergency-only modes** must be labeled as such; avoid background tracking for non-emergency features without separate review.

## Camera / microphone

- **No ambient capture** without an obvious on-screen indicator and user action.
- **Retention** — define minimum retention; avoid shipping raw media to unknown endpoints.

## Consent

Use plain language; link to the relevant product privacy note when one exists.
