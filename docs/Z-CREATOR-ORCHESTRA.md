# Z-Creator Orchestra — Creative Media Direction Engine

**Tagline:** _The Sanctuary media director for songs, films, podcasts, books, images, and legacy storytelling._

**Purpose:** Define **Z-Creator Orchestra** as a **draft-only**, **human-gated** creative layer for Z-Sanctuary: it aligns tone with journey and receipts, proposes media briefs and creative outputs, and **never** publishes or ships without approval. This document is **doctrine-first**; no automation of generation or publishing is assumed or required by this file.

**Authority:** When unsure where a capability belongs or what may be read vs written, check [Z-HIERARCHY-CHIEF-AND-OBSERVER-VIEW.md](Z-HIERARCHY-CHIEF-AND-OBSERVER-VIEW.md). Ethics and vault boundaries: [vault/Vault_Policy_Sheet.md](vault/Vault_Policy_Sheet.md), [vault/Vault_Access_Map.md](vault/Vault_Access_Map.md). Formulas / DRP alignment: [Z-ULTRA-INSTINCTS-AND-FORMULAS.md](Z-ULTRA-INSTINCTS-AND-FORMULAS.md).

---

## 1. Core rule

```text
AI creates drafts.
AMK-Goku / AT / JB approve.
System records.
Nothing publishes alone.
```

Outputs are **labeled** clearly when fictional, illustrative, demo-only, or not yet approved for external use.

---

## 2. Role of the orchestra (conceptual engine)

```text
Read the Sanctuary state
Understand the journey
Compose creative briefs
Draft songs / scripts / podcast outlines / books
Keep style aligned
Send outputs for human approval
Never publish automatically
```

Z-Creator Orchestra is **not** a replaceable streaming service or auto-post bot. It is a **Director’s desk inside the Sanctuary**: coherence, pacing, consent, and record-keeping—not volume.

## 3. Scope (media types covered)

| Area | Typical draft outputs |
| ----------------- | --------------------------------------------------------------------------------------------------- |
| **Songs** | Lyrics, melodic notes (non-binding), thematic briefs |
| **Films / video** | Treatments, scene lists, dialogue drafts, pacing notes |
| **Podcasts** | Episode arcs, outlines, cold opens, segment titles |
| **Books** | Chapter beats, synopsis, thematic spine, sample passages |
| **Images** | Shot lists, composition briefs, mood boards (describe-only unless separate image pipeline approved) |
| **Social posts** | Copy drafts, thread skeletons |
| **Pitch visuals** | Deck narrative, slide-by-slide copy, refrain from claiming live metrics not in receipts |
| **Documentaries** | Ethical framing, interview themes, factual vs interpretive labeling |

Anything public-facing inherits **Responsible Use** posture and must not pressure, shame, or extract.

## 4. Allowed inputs (“may read”)

These are **categories** Cursor or future tooling may ingest **when policy allows**, not blanket permission to scrape private data.

| Input | Notes |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Receipt / ledger style reports** | E.g. `ai:ecosphere`-class outputs and operator receipts—**summaries**, not secrets. See [AGENTS.md](../AGENTS.md) for hub commands; treat reports as observational. |
| **Commander-adjacent state** | Narrative snapshot of calm vs heavy phases (from sanctioned reports/UI copy), **not** unapproved execution. |
| **Z-PI and project identities** | Public-safe project names and stories as recorded in sanctioned registries—not invented bios. |
| **Z-Product Sovereign Registry (Lite)** | Product story lines that already exist as planning JSON/docs. |
| **Z-ANIMA / Connection Tree philosophy** | Emotional themes ([Z-CONNECTION-TREE-PHILOSOPHY.md](Z-CONNECTION-TREE-PHILOSOPHY.md))—**themes**, never private lives. |
| **Twin Roots** | Cultural memory metaphors approved in docs/mock panels—not personal vault. |
| **Z-Canvas Root Map** (when present) | Visual story map narratives as already documented — read-only glue for coherence. |

**Forbidden inputs without explicit gate:** Sovereign Vault raw material ([Vault_Access_Map.md](vault/Vault_Access_Map.md)); credentials; private individual health/history unless already cleared for creative use in writing; unpublished third-party copyrighted works as paste-to-rewrite “style donors.”

## 5. Allowed outputs (“may generate”)

- Creative **briefs** (one-page director notes).
- **Draft** lyrics, scripts, outlines, book synopses, podcast rundowns.
- **Style guides** (“voice of this era” for Sanctuary fiction vs documentary vs kids’ lane)—always marked draft.
- **Change logs** tying a draft revision to receipt IDs / dates when operational memory exists.

Every deliverable carries: **Draft — not approved**, **fiction/demo** where applicable, and **who must sign off**.

## 6. Creative memory vault (concept — not vault access)

“Creative Memory Vault” here means **a future, explicit store of approved beats and tonal decisions** separate from Sovereign Vault.

- Holds **summaries**: approved taglines, era names, sanctioned metaphors—not raw personal dossiers.
- Is **written only after human approval**, not mined automatically from chat logs.
- If implemented later, follows Folder Manager / governance paths; does **not** override [Vault_Policy_Sheet.md](vault/Vault_Policy_Sheet.md).

Until such a construct exists on disk/registry, treat this section as **doctrine naming only**.

## 7. Director roles (logical lanes)

Roles are hats for Cursor or collaborators; **one Director’s note** may combine hats.

### Song Director

- Tracks emotional arc vs ANIMA / Connection Tree posture.
- Proposes hooks, refrain structure, BPM mood—lyrics stay draft until signed.

### Film Director

- Table reads and scene intents; pacing vs Commander “phase” metaphors **without claiming real ops footage**.

### Podcast Producer

- Hook, acts, timestamps outline; disclaimers where advice-adjacent; no fake guest quotes.

### Book Architect

- Acts, pacing, thematic spine; sample prose clearly marked illustrative.

### Visual Art Director

- Mood, palette, symbolism—safe for minors when kids’ lanes exist; avoids exploitative glamour.

### Trailer / Pitch Director

- Stakes and promise **without lying** about product maturity; ties to Sovereign Registry “truth lane” wording.

## 8. Review gates (non-negotiable)

| Gate | Requirement |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Ethics / DRP** | Output must fit non-harm and compassion-aligned posture referenced in Sanctuary docs and Ω / DRP material; no gratuitous cruelty, coercion, or humiliation narratives. |
| **No impersonation** | Do not pose as named real people unless explicitly authorized and labeled. |
| **No copyrighted style cloning** | Inspiration is thematic; **do not** paste or mimic identifiable third-party lyric/dialogue to dodge copyright/trademark/trade-dress claims. Clear room tone is original. |
| **No Vault leakage** | No Sovereign Vault or private medical/family arcs in drafts unless a human has explicitly authorized that framing. |
| **Human approval** | AMK-Goku / AT / JB (or delegated author) approve before “final” labeling or downstream sync. |
| **Publish** | Separate explicit step; **no automatic** post, upload, or store billing. |
