# Z-STILLNESS-LEARN-1 — Stillness Mode & AI learning pathway

Formal ID: **Z-STILLNESS-LEARN-1**.

## Purpose

Give every Z-AI-facing lane a **safe default when there is no active task**: stay aligned by **reading, mapping, reflecting, and reporting** — not by inventing work, editing code, or running background automation.

Phase 1 ships **doctrine documents, JSON registry, sample posture rows, read-only validator, reports, and a dashboard overlay** only. There is **no** stillness daemon, cron learner, autonomous agent pool, provider fine-tuning, or silent self-upgrade.

## Golden law

```text
Idle AI ≠ permission to act.
Stillness Mode ≠ background automation.
Learning roadmap ≠ self-upgrade authority.
Understanding ≠ execution.
AMK-Goku owns sacred moves.
```

Signals follow hub traffic semantics where they apply:

- **GREEN** — read-only alignment and learning summaries.
- **YELLOW** — role overlap or taxonomy ambiguity; deepen docs before widening teams.
- **BLUE** — policy or ethics question; AMK decides before any new tooling lane.
- **RED** — autonomous execution, deploy, providers, secrets, live connectors, or self-upgrade patterns — blocked.

Reminder: **GREEN ≠ deploy.**

## Stillness vs action (quick reference)

| When idle, an AI may… | Safe in Phase 1? | Notes |
| --------------------------------- | ---------------- | ------------------------------ |
| Read doctrine and registry docs | Yes | Improves alignment |
| Summarize or align in text | Yes | Report-only outputs |
| Note missing links or doc gaps | Yes | Operator pastes or tickets |
| Propose learning paths (no edits) | Yes | Suggestions only |
| Compare roles / overlap | Yes | Supports fusion / routing maps |
| Edit code, merge, deploy | No | Sacred / task-gated |
| Spin background agents or cron | No | Stillness is not automation |
| Call providers or train models | No | Forbidden stillness escalation |

## Learning levels (pathway)

1. **Identity** — Which team, which project, allowed vs forbidden outputs.
2. **Ecosystem map** — Hub vs Labs capsules, authoritative docs vs evidence reports.
3. **Laws** — 14 DRP, Turtle Mode, traffic semantics, GREEN ≠ deploy.
4. **Routing** — MOD-DIST, XBUS, SEC, Pattern-Safe, Replica Fabric, Ultra MAGE, IDE fusion, AMK SYNC (see router docs).
5. **Formula awareness** — MAGE codex, Replica Fabric, Pattern-Safe, NUMEN (vocabulary only).
6. **Stillness output** — Only the report types listed under **Allowed Phase 1 outputs**.

## Allowed Phase 1 outputs

An idle AI may **describe** these artifact types only (human or ticket carries them forward):

- `learning_summary`
- `alignment_note`
- `missing_doc_suggestion`
- `role_clarity_suggestion`
- `risk_observation`
- `decision_packet_draft` (evidence for AMK — not approval)
