# Zuno memory and knowledge layer

**Purpose:** Separate **types of memory** so orchestration does not collapse vault truth, creative drafts, and audit receipts into one undifferentiated blob.

## Knowledge sources (already in hub)

Zuno-shaped awareness should draw from **evidence-bearing** artefacts:

| Source | Role |
| ----------------- | ---------------------------------------------------------------------------------- |
| Registries | `data/z_master_module_registry.json`, manifests, product registries |
| Markdown doctrine | `docs/**`, AI Builder pack, safety index |
| Generated indexes | `docs/Z_SANCTUARY_MODULE_INDEX.md`, engines, safety indexes |
| Reports | `data/reports/**` — Zuno coverage, Phase 3 plan, CAR², guardian, comm-flow |
| Dashboard | Read-only panels and registries ([dashboard/README.md](../../dashboard/README.md)) |
| Git history | Forensics only; not automatic “memory” without operator intent |

## Memory types (conceptual)

| Type | Purpose | Boundaries |
| --------------- | ------------------------------- | ----------------------------------- |
| Project memory | Repo layout, modules, scripts | Registry IDs; no invented paths |
| Formula memory | Z-formula frames and docs | Frameworks, not guaranteed outcomes |
| Safety memory | DRP, build rules, boundary docs | Overrides speculative outputs |
| User memory | AMK-approved prefs and goals | Human-curated; no covert profiling |
| Audit memory | Receipts, verify outputs, gates | Append-only mindset |
| Creative memory | Songs, stories, branding drafts | IP and consent aware |

## Alignment with other protocols

- **Z-SAGE:** observes **signals** from reports — not raw vault crawl ([Z-SAGE-CORE-OBSERVATION-PROTOCOL.md](../Z-SAGE-CORE-OBSERVATION-PROTOCOL.md)).
- **Z-RPWC:** cross-project sharing stays **summary-only** unless chartered ([Z-RPWC-REPLICABLE-POWER-WITHOUT-CHAOS.md](../Z-RPWC-REPLICABLE-POWER-WITHOUT-CHAOS.md)).
- **Sovereign vault:** read-only sincerity frequency per vault policy — never merge into generic “AI memory” without gates.

## Implementation stance

**Phase A:** This doctrine + explicit pointers in orchestrator roadmap.
**Phase B:** Optional read-only index scripts (local scan → report only) — see master roadmap Phase 6.

## Forbidden

- Treating markdown doctrine as executable permission.
- Storing secrets or vault payloads in “convenient” shared memory stores.
