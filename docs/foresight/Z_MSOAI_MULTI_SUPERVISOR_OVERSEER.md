# Z-MSOAI — Multi-Supervisor Overseer AI

**Status:** Doctrine only (Phase Z-FUTURE-1). **Review roles**, not judges with authority.

## Intent

Z-MSOAI describes a **council-style review layer**: multiple **overseer roles** inspect the **same** future-roadmap or scenario from different angles, producing **structured disagreement and alignment notes** for humans — not a single AI verdict.

Overseer lenses (illustrative, expandable under charter):

| Lens | Question posture |
| ---------------- | ------------------------------------------------------------ |
| Ethics / dignity | Who is harmed or excluded if this branch wins? |
| Safety | What fails closed? What needs kill-switch or rollback? |
| Economics | Who pays, who benefits, what is subsidized? |
| Ecology | Long-horizon resource and resilience stress (where relevant) |
| Education | Literacy, access, manipulation risk |
| Law / governance | Terms, privacy, AI Act posture, entitlement honesty |
| Accessibility | WCAG/EAA-style readiness, not fake certification |
| Technology | Honest capability vs entitlement separation |

## Alignment with responsible AI framing

Risk-management frameworks (e.g. **NIST AI RMF**) and principles for **trustworthy AI** (e.g. **OECD AI Principles**) inform **how we talk about review** — this document does **not** invoke those bodies as endorsers of Z-Sanctuary outputs and does **not** add API or runtime integration.

## Hard boundaries

| True | False |
| -------------------------------------------------------------- | -------------------------------------------------- |
| Overseers are **checklists and personas** for human-led review | Overseers replace lawyers, safety officers, or AMK |
| Output is **advisory** | Output is binding policy or deploy approval |
| Multiple angles reduce **single-model blind spots** | Council vote auto-executes in production |
