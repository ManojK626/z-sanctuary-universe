# Z-PPPFA — Pre-Pattern Present-to-Future Analyzer

**Status:** Doctrine only (Phase Z-FUTURE-1). **Not** a prediction engine, oracle, or certainty generator.

## Intent

Z-PPPFA names a **disciplined way to think** from **past patterns** and **present signals** toward **plausible futures** — so the ecosystem can **pre-analyze possibilities** before repeating avoidable mistakes. This aligns with **strategic foresight** practice: helping decision-makers work with **risk and uncertainty**, not claiming foreknowledge of outcomes. See UNDP and multilateral long-horizon framing (e.g. UN “Our Common Agenda”) as **external context** — the hub implements **documentation and registry links only**.

## What Z-PPPFA does (in principle)

| Step | Description |
| --------- | ----------------------------------------------------------------------------------- |
| Observe | Past patterns (documentation, receipts, incidents, structural drift) |
| Read | Present signals (traffic, SSWS, LAWGRID, Zuno coverage, CAR² hints) |
| Map | **Scenarios** — multiple branches, not a single forecast |
| Detect | Recurring failure modes (entitlement confusion, deploy optimism, capability creep) |
| Recommend | **Safer pathways** as **options for human review** — never as automatic instruction |

## Wording we use

- Pre-pattern analysis, present-to-future mapping, scenario, branch, signal, risk review, human decision required.

## Wording we avoid

- Guaranteed prediction, prophecy, oracle, certainty, inevitability, automatic civilization steering.
