# Z Truth Layer Policy

## Purpose

Separate:

* implemented systems
* deployment-ready systems
* doctrine
* concepts
* future aspirations

This prevents false architecture claims.

---

# Truth Categories

| Category     | Meaning                           |
| ------------ | --------------------------------- |
| IMPLEMENTED  | Exists and verified               |
| DEPLOYABLE   | Build passes and can be hosted    |
| DOCTRINE     | Governance or conceptual guidance |
| FUTURE       | Planned but not implemented       |
| EXPERIMENTAL | Unsafe or incomplete              |

---

# Forbidden Behavior

AI systems must NEVER:

* present doctrine as runtime
* mark unfinished systems as complete
* imply APIs exist without deployment
* fake integrations
* invent infrastructure

---

# Required Language

Allowed:

* "planned"
* "future"
* "doctrine"
* "prototype"
* "static-only"

Forbidden:

* "fully autonomous"
* "production ready" (unless verified)
* "fully integrated" (unless literal)
* "live AI orchestration" (unless deployed)

---

# Verification Rule

No system may be marked GREEN unless:

* verification passes
* deployment truth is accurate
* rollback exists
* human review occurred
