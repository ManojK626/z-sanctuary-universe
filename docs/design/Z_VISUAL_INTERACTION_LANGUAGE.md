# Z Visual Interaction Language (Z-VIL) — Hub pointer

## Status (QX-1A)

**Doctrine and metadata only.** There is no shared npm package, no automatic runtime coupling, and no requirement that other projects inherit Z-QUESTRA UI or components.

## Canonical seed

The living **implementation seed** and pillar table live in Z-QUESTRA:

- [z-questra/docs/Z_VISUAL_INTERACTION_LANGUAGE.md](../../z-questra/docs/Z_VISUAL_INTERACTION_LANGUAGE.md)

Hub operators and sibling projects should treat that file as the **reference** for color realms, comfort modes, motion limits, photophobia-aware patterns, age-mode themes, and local-only receipt posture — aligned with Phase 2.3 visual structure docs inside `z-questra/docs/`.

## How other projects should reuse Z-VIL

| Approach | Allowed now |
| ------------------------------------------------------------------------------------------ | ----------: |
| Read the seed doc and **copy discipline** (naming, accessibility habits, non-predatory UX) | Yes |
| Link to Z-QUESTRA docs from operator manuals | Yes |
| Import a **shared component package** or embed runtime without charter | No |
| Assume UI parity or shared tokens across repos | No |

Formal expansion (packages, embeds, bridges) stays behind **charter** and **14 DRP / operational gate** review, consistent with [z-questra/docs/Z_QUESTRA_PORTABILITY_AND_BRIDGE_POLICY.md](../../z-questra/docs/Z_QUESTRA_PORTABILITY_AND_BRIDGE_POLICY.md).
