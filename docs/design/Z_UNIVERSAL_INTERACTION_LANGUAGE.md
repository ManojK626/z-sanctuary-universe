# Z Universal Interaction Language (Z-UIL)

**Status:** Doctrine and architecture text only — no runtime coupling, no automatic cross-repo UI, no pricing merge, no deployment authority.

**Scope:** Defines how **all Z-Sanctuary-related projects** can share **one recognizable ecosystem identity** while keeping **commercial, emotional, operational, and legal boundaries** explicit.

**Relationship to Z-VIL (visual layer):**

- **[Z_VISUAL_INTERACTION_LANGUAGE.md](Z_VISUAL_INTERACTION_LANGUAGE.md)** is the hub pointer to **Z Visual Interaction Language (Z-VIL)** — realms, comfort modes, motion discipline, photophobia-aware patterns. Canonical implementation discipline still lives with the **Z-QUESTRA** seed cited there unless a chartered package exists.
- **Z-UIL** is the broader **interaction and governance constitution**: where users think they are, what they may do, how capabilities differ from **entitlements**, how benchmarks and queues are worded, and how **navigation** stays honest across projects.

If Z-VIL and Z-UIL ever conflict on safety or accessibility, **accessibility + DRP non-harm** win; unresolved product conflicts go to **[Z-HIERARCHY-CHIEF-AND-OBSERVER-VIEW.md](../Z-HIERARCHY-CHIEF-AND-OBSERVER-VIEW.md)** and human oversight.

| Goal | Outcome |

| Long-term safety | **Shared capability** never silently becomes **shared entitlement** |

## 2. Shared interaction laws

These laws apply cross-project unless a scoped exception is **written**, **approved**, and **indexed**.

1. **Truth in surface:** Labels state what the surface is (demo, cockpit, navigator, readiness view). No panel implies endorsement, privileged advice, or auto-approval.
2. **Receipts optional, claims careful:** Evidence and reports attach to artefacts; GREEN-style language never implies worldwide compliance without counsel.
3. **Human gates for sacred moves:** Merge, deploy, payment, privileged legal storage, vault admin, autonomous execution — gated per hub rules (see `AGENTS.md`, Turtle Mode, release control).
4. **No camouflaged cross-sell:** A familiar icon or motif must not imply “you already paid for Project B.”
5. **DRP as moral floor:** Interaction copy and automation respect **14 DRP** framing (see swarm law docs); ambiguity defaults to non-harm and human review — not maximal automation.

## 4. Visual language doctrine

- **Tokens:** Prefer shared naming for colours, spacing, radii — but **consumption per repo** stays opt-in unless a chartered package ships.
- **Department identity:** Iconography and captions should signal **department** (governance versus education versus playful), not impersonate authority.
- **Panel glow:** “Personality glow” reinforces mood; benchmark and legal-facing panels lean **restrained**, **readable**, disclaimer-forward.
- **Typography:** Honour accessibility baseline (sizes, contrast, avoid tiny legal copy). Typography tokens may defer to **Z-VIL age/comfort modes** where implemented.

---

## 5. Entitlement separation rules (critical)

### 5.1 Core rule

> **Shared patterns, shared lore, shared tooling ethos — optional.**
> **Paid tier, quotas, vaults, API keys, legal review artefacts, deploy permissions — NEVER implied by resemblance alone.**

### 5.2 Must always be distinguishable

| Domain | Separation mechanism |
| ---------------------- | ---------------------------------------------------------------------------------------------------------- |
| Pricing / SKU | Explicit price pages, SKU tables, entitlement registries (`z_pc_root_projects`-style manifests where used) |
| Premium AI usage | Capability flags + human-readable “not included elsewhere” clauses |
| Private vault | Never reachable from unrelated project skins without SSO + policy |
| Legal review artefacts | Locked to Z-Legal / counsel workflows |
| Deploy | Only named environments and roles; dashboards display **intent**, not webhook execution |

## 7. Accessibility baseline

Minimum baseline for any public-ish surface claiming Z-UIL conformance at “draft readiness”:

- Keyboard reachability for primary navigation and tab strips.
- Visible focus indicators; sensible tab order.
- Respect `prefers-reduced-motion` for ornamental effects.
- No necessary information encoded only by colour (pair with icons/text).
- Legal and benchmark panels: plain-language disclaimers adjacent to coloured signals.

Aligned with **Z-LEGAL-BENCH-1**:

| ------------------------------------------------------ | -------------------------------------------------------- |

## 13. DRP interaction layer

User-facing and operator flows should state when:

- an action is **advisory** only,
- a human must confirm,
- a child-safety, health, or emergency path is **out of scope**,
- an AI output is **not** a substitute for professional services.

Swarm and agent rules remain authoritative implementation detail; Z-UIL is the **plain-language** mirror for design and copy.

## 14. Launch consistency rules

1. **Staging language:** “Internal / staging / preview” never displayed as “live production” without green-path receipts and release control.
2. **Parallel legal review:** Public marketing lags **minimum** terms/privacy/disclaimer package when counsel says so (see benchmark deployment row).
3. **Feature flags:** Optional capabilities ship dark until doctrine and registry entries exist.
4. **Rollback story:** Every launch description names how to revert (docs + operator paths).
