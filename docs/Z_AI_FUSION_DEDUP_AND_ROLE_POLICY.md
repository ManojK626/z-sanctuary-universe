# Z-AI Fusion — deduplication framing + role policy

Companion to **Z-AI-FUSION-MAP-1**.

## Principle

Healthy ecosystems tolerate **overlap** — the failure mode is **unowned overlap** plus **silent authority drift**. Governance states **intent**: who leads, who supports, what is forbidden, whether dashboard rows should consolidate **by AMK**.

## Patterns

| Pattern | Meaning | Default posture |
| ------------------ | -------------------------------------------------- | -------------------------------------- |
| Exact duplicate | Same mandate + same outputs | `BLUE_AMK_REVIEW` or `ALIAS_TO_LEAD` |
| Near duplicate | Similar mandate, different receipts | `LEAD_SUPPORT` or `MERGE_DOCS_ONLY` |
| Sibling lane | Same domain, different artefact depth | Usually `KEEP_SEPARATE` with hierarchy |
| Conflicting advice | Signals disagree | Always `BLUE_AMK_REVIEW` |
| Unsafe overlap | Merged forbidden lanes expose deploy/payment/leaks | `RED_BLOCKED` |

## Locked law

```text
Fusion map ≠ auto-merge.
Overlap score ≠ deletion.
Lead AI ≠ sacred authority.
Dashboard grouping suggestion ≠ runtime execution permission.
GREEN ≠ deploy.
BLUE requires AMK.
RED blocks movement.
AMK-Goku owns sacred moves.
```
