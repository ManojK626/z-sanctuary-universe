# Z-MAOS — Z-Sanctuary Multi-Project AI Operating Supervisor (Charter)

**Purpose:** Define **Z-MAOS** as an **operator-first coordination layer** for the full Z-Sanctuary multi-project ecosystem: launcher context, workspace health, extension readiness, safe status checks, mini-bot routing, consent gates, and friction reduction—**without** silent merge, deploy, publish, delete, or unsanctioned coupling.

**Authority:** [Z-HIERARCHY-CHIEF-AND-OBSERVER-VIEW.md](../Z-HIERARCHY-CHIEF-AND-OBSERVER-VIEW.md). Turtle / Overseer discipline: [AGENTS.md](../../AGENTS.md).

---

## 1. Mission (what Z-MAOS is)

```text
One operator layer for the full Z-Sanctuary Universe:
multi-project launcher
AI bot routing
workspace health
extension readiness
cycle checks
module status
consent gates
friction removal
cross-project clarity
```

Z-MAOS **coordinates**. It does **not** replace AMK-Goku, Z-OSHA/Overseer release gates, or product-specific cockpits.

---

## 2. Core principle: cross-link, don’t cross-contaminate

```text
Z-Sanctuary = central universe / governance / memory / trust layer
XL2 = standalone product, referenced but not coupled
Other projects = registered satellites, not blindly merged
```

The supervisor may **know** paths, phases, and allowed commands **as declared in registry JSON**—it must **not** import or rewrite satellite code unless a **separate charter** explicitly allows it ([CROSS_PROJECT_LINKING_POLICY.md](CROSS_PROJECT_LINKING_POLICY.md)).

---

## 3. Safe behaviour (must)

- Launch **approved** read-only checks (or print commands for the operator to run).
- Report missing requirements (extensions, docs, reports).
- Route understanding to **mini-bot doctrine** ([MINI_BOT_DISPATCH_RULES.md](MINI_BOT_DISPATCH_RULES.md)).
- Surface **consent-gated** actions ([AMK_CONSENT_GATES.md](AMK_CONSENT_GATES.md)).

---

## 4. Forbidden behaviour (must not)

```text
It does not silently merge, deploy, publish, delete, or connect sensitive systems.
```

Also: no auto-install of VS Code extensions; no external connectors without consent; no weakening pause/NAS/sandbox gates; no Vault raw exposure.

---

## 5. Autonomy default

See [AUTONOMY_LEVELS.md](AUTONOMY_LEVELS.md). **Default L2** (task routing / command suggestions). **L3** only supervised after AMK approval. **L4/L5** off unless a future charter changes that.
