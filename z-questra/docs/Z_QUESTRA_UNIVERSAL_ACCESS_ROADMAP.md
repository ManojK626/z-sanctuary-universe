# Z-QUESTRA — Universal Access Roadmap

## Vision

Make Z-QUESTRA **discoverable** and **portable in principle** across Z-Sanctuary: operators know what it offers and how to reuse it safely — **without** silent coupling.

## Phase ladder (high level)

| Phase | Focus | Risk |
| -------- | -------------------------------------------------------------- | ------------------ |
| **QX-1** | Capability manifest + portability docs + hub catalog alignment | Low |
| **QX-2** | Further hub catalog splits (sub-rows) if needed | Low |
| **QX-3** | Static launch card copy on hub dashboard (still links only) | Low |
| **QX-4** | Shared visual language package proposal | Medium |
| **QX-5** | iframe/embed **research** doc (no default embed) | Medium |
| **QX-6** | Local bridge prototype (no cloud) | Medium |

## Current truth

- **QX-1 complete** when manifest + policy docs + green receipt exist and tests pass.
- **Bridge:** `reference_only` in manifest until charter says otherwise.
- **Memory:** opt-in inside app only; cross-project memory remains **future gated**.
