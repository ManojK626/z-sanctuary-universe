# AI Builder Context (Labs)

## Purpose

This lab runs doctrine-first module incubation. Phase 1 deliverables are docs, registries, validators, and read-only reports.

## Z-Logical Brains framing

- Use non-clinical language: learning pathways, visual reasoning, adaptive education, and self-reflection.
- Do not position this module as medical diagnosis, therapy, treatment, or guaranteed improvement.
- Treat the source PDF as a concept seed, not production code.

## Hard limits for Phase 1

- No backend runtime services
- No user accounts or authentication flows
- No cloud sync or provider integration
- No vault runtime or production encryption claims
- No child data collection by default
- No voice, camera, GPS, or hidden tracking

## Z-NUMEN framing

- Companion to **Z-Logical Brains**: Logical Brains = learning pathway; NUMEN = **pattern literacy** (repetition, symbolic geometry bands, Zero Core grounding).
- **Z-NUMEN-LB-1:** reference-only pairing is captured in `data/z_numen_logical_brains_companion_map.json` and `docs/Z_NUMEN_LOGICAL_BRAINS_COMPANION_MAP.md` — **no runtime merge**, `runtime_coupling: false`.
- Use **educational / perception-awareness** language only. Do not claim hidden control, supernatural proof, diagnosis, therapy, or prediction certainty.
- Formulas in `data/z_numen_symbol_registry.json` are **documentation-only**; the validator uses simple local heuristics, not profiling engines.
- Future 3D / canvas visuals are **future-gated** in the registry; Phase 1 is docs + JSON + read-only reports.

## Verification focus

Phase 1 is complete only when registry JSON, safety JSON, and validator reports align and signal safe non-clinical planning. Run `npm run z:numen` after registry edits; combined lab check: `npm run verify:local`.
