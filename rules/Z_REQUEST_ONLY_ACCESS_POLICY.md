# Z-Request-Only Access Policy

## Purpose

Protect code, models, modules, architecture, and sensitive operational knowledge from unauthorized extraction.

## Default Access Mode

- `public`: simple facts only, read-only, privacy-first.
- `deep details`: restricted, request-only, governance-reviewed.

## Protection Rules

- No selection/copy on protected surfaces by default.
- No auto-download of internal artifacts from public views.
- No direct public indexing of private module structures.
- No automated ingestion of external research into core logic.

## Request Workflow

1. User submits structured request via trust portal.
2. Request is reviewed by governance + AI guardrails.
3. Approved scope is disclosed minimally and time-bounded.
4. Access decision is logged to audit trail.

## Guardrail Statement

Z-Sanctuary does not process or disclose protected internal knowledge unless explicitly approved through the request workflow.
